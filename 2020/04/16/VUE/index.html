VUE项目图片展示

Posted on 2020-04-16

使用 Vue 进行图片展示项目的开发 使用 vue-cli 脚手架工具及进行开发

1、首先全局装 vue-cli 使用 npm install -g @vue/cli

2、检查 vue-cli 版本的命令 vue –version

3、遇到 ‘webpack-dev-server’ 不是内部或外部命令，也不是可运行的程序 的问题，原因一般是一些依赖没有成功安装导致的

4、解决：将 node_modules 文件夹删除之后 再重新 npm install 安装一次

初始化项目 vue init webpack travel

单页面应用和多页面应用的区别 优缺点

单页面应用：

首屏加载时间相对较长（一次性将页面的所有数据请求回来）缺点

SEO 效果差 缺点

页面切换速度快 （不会再重复进行 http 请求） 优点

多页面应用

首屏加载时间短 只发送当前页面的http请求 优点

SEO效果好 优点

页面切换速度相对较慢 （需要发送 http 请求） 缺点

需要引入 reset.css 和 border.css 文件，放入 src 目录下 assets 文件夹下style文件夹

reset.css 用来解决不同手机样式显示不一样问题

border.css 用来解决移动端 1px 的问题

引入 fastclick 来解决 click 事件 300ms 延迟问题 npm install fastclick

需要在 src 目录下的 main.js 中引入相应文件（reset.css border.css fastclick）

import Fastclick from ‘fastclick’;

import ‘./assets/style/reset.css’;

import ‘./assets/style/border.css’;

import ‘./assets/style/iconfont.css’;

需要使用到 iconfont 图标库 也需要将对应的 css 文件引入

iconfont.css文件 放在 src 目录下的 assets 文件夹 style 里

从 iconfont 官网下载下来的压缩包 需要用到的字体文件后缀有四个 .ttf .svg .eot .woff

iconfont使用方法

一般使用 或者 标签来承载图标，给 / 添加类名 iconfont

到 iconfont 官网复制对应图标的唯一标识  填写在内容中



每次增加图标都要重新下载一次并且重新替换 5个文件（.ttf .svg .eot .woff iconfont.css）

使用 gitee 来管理项目，在gitee上创建项目

git使用到的命令

1、git add . 将改动的文件加入到临时仓库

2、git commit -m “提交的描述” 将临时仓库的文件加入仓库

3、git push 将仓库中的文件推送到gitee上同步

4、在 gitee 上创建分支，开发都是在分支上开发，不是在主分支开发（默认的是 master）

5、在本地使用 git pull 同步远程仓库和本地的变化

6、git checkot 切换分支

7、git branch 查看所有分支

8、git branch -d 删除分支 是要删除分支的名称

9、git merge 合并某分支到当前分支，一般是切换到 master 分支再合并上去

轮播图使用 swiper

vue轮播插件 vue-awesome-swiper

在组件的 style 中写 scoped 来封闭作用域，不影响外界作用域

如果当前组件使用到了全局样式，需要用 >>> 来穿透或者 /deep/ >>> 有一点点问题

设置元素的宽高比，将 height 设置为 0，用 padding-bottom 设置百分比（相对于父级的宽度来决定，例如父级宽度 100px ，子级padding-bottom: 50%; 那就是 50px）屏幕变宽了也能保持高度跟随变化
